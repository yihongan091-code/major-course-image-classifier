# AI 看图理解接口说明

网页会向你填写的“接口地址”发送 `POST` 请求。

## 请求格式

```json
{
  "major": "计算机科学与技术",
  "courses": ["数据结构", "操作系统", "数据库系统"],
  "fileName": "课堂照片.jpg",
  "image": "data:image/jpeg;base64,...",
  "instruction": "请理解图片中的教材、课件、题目或课堂内容，并判断它最可能属于当前专业的哪一门课程。"
}
```

如果填写了访问密钥，请求头会包含：

```http
Authorization: Bearer 你的密钥
```

## 响应格式

接口返回 JSON：

```json
{
  "course": "数据结构",
  "chapter": "二叉树遍历",
  "questionType": "算法题",
  "ocrText": "请写出二叉树的前序遍历结果",
  "summary": "图片中出现二叉树遍历、节点和递归算法内容。",
  "keywords": ["二叉树", "遍历", "递归"],
  "confidence": 94
}
```

字段也支持中文别名：

```json
{
  "课程": "数据结构",
  "内容概述": "图片中出现二叉树遍历、节点和递归算法内容。",
  "关键词": ["二叉树", "遍历", "递归"]
}
```

## 说明

- `course` 应尽量从 `courses` 列表中选择。
- `chapter` 是章节或知识点，`questionType` 是题型或内容类型。
- `ocrText` 是从图片中提取的关键文字。
- `summary` 会显示在识别结果里。
- 如果 AI 接口不可用，网页会自动退回本地关键词和视觉线索识别。
